403 - Access Denied

Something Went Wrong!

Security Check Failed

The request could not be processed because a security token validation failed. This is typically caused by a missing or incorrect Cross-Site Request Forgery (CSRF) token.

Primary Cause

The CSRF token in your request was missing or incorrect. This could happen because:

  • Your browser's cookies are disabled or blocked
  • The page was cached or reloaded without the proper token
  • You logged in or out in another browser tab
  • The form submission didn't include the required security token
Note: CSRF tokens are rotated after login for security reasons. If you've recently logged in, please reload the form page to get a new token.

If You're a User

  • Ensure cookies are enabled in your browser
  • Refresh the page and try the action again
  • Clear your browser cache and cookies, then retry
  • Log out and log back in if the problem persists

If You're a Developer

  • Ensure {% csrf_token %} is inside your POST form
  • Verify CsrfViewMiddleware is enabled in settings
  • Check that the view passes request to template render method
  • For AJAX requests, include the CSRF token in headers
